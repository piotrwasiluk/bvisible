# Design System Specification: Linear Mission Control

## 1. Overview & Creative North Star: "The Informed Void"
The Creative North Star for this design system is **The Informed Void**. In an era of AI-driven noise, this system acts as a high-precision instrument—a "Mission Control" for technical execution. It moves beyond standard minimalism into a realm of **Functional Brutalism**, where every pixel must justify its existence.

To break the "template" look, we employ **Intentional Asymmetry**. Instead of centering content, we anchor elements to a rigid left-aligned axis, using massive whitespace to create a sense of scale and "gravity." We favor high-contrast typography and razor-sharp structural transitions over decorative flourishes. The result is an interface that feels fast, objective, and authoritative.

---

## 2. Colors: The Monochrome Spectrum
Our palette is rooted in absolute blacks and surgical grays. We use color only as a signal, never as decoration.

### Core Palette
- **Primary (`#000000`):** Used for primary actions and high-level headers. It represents the "Point of Execution."
- **Surface (`#F9F9FA`):** The default canvas.
- **Surface Container Highest (`#E2E2E3`):** Reserved for active UI states or focused panels.

### The "No-Line" Rule
Traditional 1px solid borders are strictly prohibited for sectioning large layout areas. Boundaries must be defined by **Background Color Shifts**.
* **Implementation:** Place a `surface_container_low` (`#F3F3F4`) sidebar against a `surface` (`#F9F9FA`) main content area. The 1% shift in tone is enough for the eye to perceive a boundary without adding "ink" to the screen.

### Surface Hierarchy & Nesting
Treat the UI as a series of physical layers of fine technical paper.
- **Base:** `surface`
- **Sectioning:** `surface_container_low`
- **Interactive Elements:** `surface_container_highest`
- **Nesting:** When placing a card inside a panel, the card should use `surface_container_lowest` (`#FFFFFF`) to "pop" against the gray background without requiring a shadow.

### Signature Textures
For high-priority CTAs, use a subtle vertical gradient from `primary` (`#000000`) to `primary_container` (`#3C3B3C`). This adds a "milled metal" feel that provides weight and professional polish.

---

## 3. Typography: Authority Through Scale
We use **Inter** for its neutral, objective legibility and a **Mono accent** for technical metadata.

- **Display (3.5rem / 2.75rem):** Use `display-lg` for data-heavy dashboard titles. Ensure letter-spacing is set to `-0.02em` to create a "tight," custom-engineered look.
- **Headline (2.25rem - 1.5rem):** High contrast against body text.
- **Title (1.375rem - 1rem):** Bold weights for section headers to maintain "Mission Control" hierarchy.
- **Body (1rem - 0.75rem):** Standard UI interactions. Use `on_surface_variant` (`#474747`) for secondary descriptions to reduce visual vibration.
- **Labels (0.6875rem):** Use Mono fonts for labels, timestamps, and ID strings. This signals "Raw Data" to the user, reinforcing the AI-visibility theme.

---

## 4. Elevation & Depth: Tonal Layering
We reject the heavy shadows of the 2010s. We define depth through light and opacity.

- **The Layering Principle:** Depth is achieved by "stacking." A `surface_container_lowest` card placed on a `surface_container_low` background creates a natural lift.
- **Ambient Shadows:** For floating menus (e.g., Command Palettes), use a shadow color derived from `on_surface` at 4% opacity with a 32px blur. It should feel like a "whisper" of a shadow, not a dark smudge.
- **The "Ghost Border" Fallback:** If a container requires a border for accessibility, use `outline_variant` (`#C6C6C6`) at **15% opacity**. The border should be felt, not seen.
- **Glassmorphism:** For overlays, use `surface` at 80% opacity with a `20px` backdrop-blur. This keeps the user's focus on the task while maintaining the "Mission Control" spatial awareness.

---

## 5. Components: Precision Primitives

### Buttons
- **Primary:** Background `primary` (#000000), text `on_primary` (#E5E2E3). Square-ish with `DEFAULT` (0.5rem) or `md` (0.75rem) corner radius.
- **Secondary:** `surface_container_highest` background. No border.
- **Ghost:** No background, `primary` text. Use for low-priority utility actions.

### Input Fields
- **Execution State:** Use `surface_container_lowest` for the background and a 1px `outline_variant` border.
- **Focus State:** The border transitions to `primary` (#000000). No "glow" or "halo" effects.

### Cards & Lists
- **No Dividers:** Forbid the use of divider lines. Separate list items using `spacing-4` (0.9rem) or by alternating background tones (`surface` vs `surface_container_low`).
- **Technical Chips:** Small, mono-type labels with a `secondary_container` background. Used for status tags or metadata.

### Added Component: "The Pulse"
A small (8px) circular indicator using semantic colors (green/amber/red). Place this next to system headers to indicate "Live AI Status." Use a slow, 2-second opacity pulse animation to represent a "living" system.

---

## 6. Do’s and Don’ts

### Do:
- **Use "Data-Density":** Don't be afraid of small text for technical details. It reinforces the "Pro" feel.
- **Anchor to the Grid:** Use the 8pt grid (`spacing-8`, `spacing-16`) religiously. Alignment is the only decoration we allow.
- **Embrace Whitespace:** Large "voids" of `surface` color allow the user's eye to rest between complex data sets.

### Don't:
- **Don't use "Soft" Grays:** Avoid warm grays or beiges. Keep the palette strictly cool and neutral.
- **Don't use Rounded Buttons:** Stay within the `0.5rem` (DEFAULT) to `0.75rem` (md) range. Fully rounded "pill" buttons feel too consumer-soft for this system.
- **Don't use Shadows for Hierarchy:** If you need to separate two sections, change the background color of one before you reach for a shadow.
